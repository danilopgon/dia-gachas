# Día de Gachas — Design Context

## Design Context

### Users
Casual Spanish users across all of Spain, though the soul of the app is Castilla-La Mancha. They open it out of curiosity or habit — usually to laugh, share with friends, or settle the eternal debate of whether today merits a bowl of gachas. The interaction is quick, lighthearted, and very shareable. There's no utility pressure; the delight *is* the point.

### Brand Personality
**Chanante · Divertido · Regional**

"Chanante" is the north star: the absurdist, deadpan humor of CLM Comedy Collective — unexpected non-sequiturs, dry cultural references, anti-climactic punchlines. The app shouldn't try to be clever; it should be genuinely weird and funny in a very Spanish way. "Regional" means it feels rooted — in the landscape, the food, the vernacular — not generic. It's a love letter to a place.

### Aesthetic Direction
The existing hand-illustrated countryside backgrounds define the visual mood and must remain the reference point: warm earthy tones (terracotta, sage green, cream sky), soft painterly depth, a rural Castilian landscape at golden hour.

- **Primary palette:** Terracotta/rust (`#e15b2e`) as the action color; warm creams and muted orange-browns for surfaces
- **Accent:** Sage green (`#94aa8b`) for secondary moments
- **Typography:** PPWoodland Bold (headings) — bold, artisanal, slightly rough. Alegreya Sans (body) — humanist, warm, readable
- **Motion:** Subtle and organic (floating sun, parallax hills via GSAP). Never frantic. Animations should feel like watching clouds drift.
- **Theme:** Light mode only. The palette is warm daylight — dark mode would break the mood entirely.
- **Anti-reference:** Avoid anything that reads as tech-startup, SaaS dashboard, or generic weather app. No gradients that look like Material Design. No cold blues or sterile whites.

### Design Principles

1. **Humor is the UX.** Copy, empty states, errors, and loading states are all opportunities for a gag. A spinner that says "Mirando al cielo..." is better than one that says "Loading...". Keep this register throughout.

2. **Rural, not rustic for its own sake.** The visual warmth should feel lived-in and authentic, not artificially folksy. Illustrations lead; UI chrome should recede and support them.

3. **Accessibility as respect.** The app has zero accessibility work done. Every future change should raise the floor: sufficient contrast on text over illustrated backgrounds, keyboard navigability, meaningful ARIA labels, focus indicators that don't fight the palette. Target WCAG 2.1 AA minimum.

4. **Mobile-first shareability.** The core loop is: check → laugh → share. The result screen must feel satisfying and screenshot-worthy on a phone. Desktop is secondary but should be delightful.

5. **Restraint in chrome, generosity in content.** The interface itself should be minimal — the illustrations and copy do the heavy lifting. Don't add UI complexity to solve problems that wit can solve.

## Tech Stack Reference
- Angular 19 (standalone components, signals, SSR-capable)
- Tailwind CSS v4 + tailwindcss-primeui
- PrimeNG v19 (Lara preset, customized to terracotta primary)
- GSAP v3 for animations
- Angular i18n for localization (Spanish)
- Light mode only (`darkModeSelector: false` in PrimeNG config)
